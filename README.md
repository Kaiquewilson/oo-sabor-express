🚀 Sobre o Projeto

O Sistema de Restaurantes foi desenvolvido com o objetivo de aprofundar conceitos de POO (Programação Orientada a Objetos), evoluindo a partir do projeto anterior Sabor Express.

Neste projeto, o foco foi criar uma estrutura mais profissional, aplicando separação de responsabilidades, organização modular e melhor manutenção do código.

⚙️ O sistema:
Lista restaurantes cadastrados
Exibe categoria, avaliação e status de atividade
Permite melhor organização das informações usando classes
Aplica validações e controle de dados
Utiliza propriedades para acesso controlado aos atributos (@property)
Usa convenção de encapsulamento com underscore (_atributo)
Organiza a execução com a função main()
Utiliza o padrão: if __name__ == "__main__":
para controle de execução do sistema

🧩 Conceitos Aplicados

Durante o desenvolvimento, foram reforçados e aplicados conceitos importantes:

Programação Orientada a Objetos (POO)
Criação e uso de classes
Encapsulamento de atributos
Uso do @property para controle de acesso
Convenção de atributos "privados" com _
Separação de arquivos por responsabilidade
Organização e escalabilidade do projeto
📁 Estrutura do Projeto

O sistema foi dividido em múltiplos arquivos, onde cada parte possui uma única responsabilidade, facilitando:

Manutenção
Leitura do código
Reutilização de componentes
Escalabilidade futura
🔄 Evolução do Projeto

Este projeto é uma evolução direta do Sabor Express, trazendo:

Melhor organização
Uso real de POO
Código mais limpo e modular
Maior proximidade com projetos profissionais
